# [Paper Read] Chain-of-Thought Prompting Elicits Reasoning in Large Language Models

- **Authors**: Google Brain (Jason Wei, Denny Zhou, et al.)
- **Year**: 2022 (NeurIPS)
- **Link**: [arXiv:2201.11903](https://arxiv.org/abs/2201.11903)
- **Tags**: #CoT #Reasoning #Prompt-Engineering #Emergence

## 1. 核心思想 (The Big Idea)

在 GPT-3 展示了强大的 **Few-Shot (少样本)** 能力后，模型在**逻辑推理 (Reasoning)** 任务（如算术、符号操作、常识推理）上依然表现不佳。

本文提出了一种简单的 Prompt 策略——**思维链 (Chain-of-Thought, CoT)**：
在 Few-shot 的范例中，不只给出“问题 -> 答案”，而是给出 **“问题 -> 中间推理步骤 (Rationale) -> 答案”**。

> **逻辑学视角：** 
> 传统的 Prompt 是从前提 $P$ 直接跳跃到结论 $Q$，模型计算的是 $P(Q|P)$，这是一个高熵的映射。
> CoT 强迫模型显式地生成推导链条 $R_1, R_2...$，将复杂的推理分解为多个简单的逻辑原子，利用 $P(Q|P, R_1...R_n)$ 来降低预测的不确定性。

## 2. 关键发现 (Key Findings)

### 2.1 涌现能力 (Emergence Ability)
CoT 并非对所有模型有效。
- **小模型 (<10B):** 使用 CoT 反而会导致性能下降（生成了错误的逻辑链，产生幻觉）。
- **大模型 (>100B):** CoT 带来的性能提升是**涌现**的。在 PaLM-540B 上，CoT 使 GSM8K（数学题）的准确率从 17.9% 飙升至 58.1%。

### 2.2 符号推理与泛化 (Symbolic Reasoning & OOD)
作者测试了 **Last Letter Concatenation**（单词末尾字母拼接）和 **Coin Flip**（硬币翻转）任务。
- **结果：** CoT 使得模型能够处理 **训练分布之外 (OOD)** 的数据。
- **意义：** 即使范例只有 2 个词的拼接，模型也能处理 4 个词。这证明模型通过 CoT 学会了**“算法逻辑 (Algorithm)”**，而不仅仅是统计分布。

## 3. 消融研究 (Ablation Studies) —— 为什么 CoT 有效？

为了证明效果不是来自于“计算量的增加”，作者设计了三个变体：

1.  **Equation Only (只给公式):** 效果差。 $\rightarrow$ 证明**自然语言**对语义理解至关重要。
2.  **Variable Computation (无关计算):** 效果差。 $\rightarrow$ 证明**推理的相关性**至关重要，单纯增加 Token 数没用。
3.  **Reasoning after Answer (先回答后推理):** 效果差。 $\rightarrow$ 证明了**因果链条 (Causality)** 的方向性，必须是“推理导致答案”，而非“答案的解释”。

## 4. 局限性 (Limitations)

- **逻辑错误：** 模型生成的思维链可能包含事实错误或逻辑断裂（虽然最后答案可能蒙对了）。
- **小模型不可用：** 对于 5070Ti 这种级别的本地小模型（如果不经过特定 SFT），直接使用 CoT Prompt 效果有限。
- **成本增加：** 输出推理过程会显著增加 Token 消耗，导致推理延迟变高。

## 🧠 Logic & Engineering Perspective (深度思考)

**1. Prompt Engineering vs Model Architecture**
这篇论文揭示了一个深刻的真理：**CoT 不需要改变模型参数**。它本质上是一种**Prompt Engineering**。它证明了 LLM 内部早已潜藏着逻辑推理能力，只需要用正确的语言形式（Language Form）去**唤醒 (Elicit)**。

**2. 从 Prompt 到 SFT (工程演进)**
虽然论文中使用的是 Few-shot Prompt，但在 2025+ 的工程实践中，我们更倾向于将 CoT **内化**。即通过 SFT（监督微调），让模型在 Zero-shot 下也能自动输出思维链（如 DeepSeek-R1, Qwen2.5）。

**3. 贝叶斯解释**
CoT 通过引入隐变量 $z$ (Reasoning Path)，将原来的求解过程 $p(y|x)$ 转化为 $\sum_z p(y|x,z)p(z|x)$。在 Greedy Decoding 下，则是寻找最优路径。

## 🔗 Related Work
- **Self-Consistency (Wang et al., 2022):** 提出通过采样多条 CoT 路径并投票来提高准确率（本文实验部分提到的改进方向）。
- **Zero-Shot CoT (Kojima et al., 2022):** 发现只需加一句 *"Let's think step by step"* 也能激发推理能力。
