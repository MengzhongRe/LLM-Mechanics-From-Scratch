### 🚀 Day 17-19：MoE 核心算子手撕路线图

#### Day 17：路由器（Router）与 Top-K 逻辑
*   **目标**：实现“大脑”部分，决定每个 Token 该去哪。
*   **执行细节**：
    1.  **Router 线性层**：定义 `self.gate = nn.Linear(dim, num_experts, bias=False)`。
    2.  **Top-K 筛选**：使用 `torch.topk` 选出前 $k$ 个专家（通常 $k=2$）。
    3.  **权重归一化**：对这 $k$ 个分数重新进行 `Softmax`，确保它们的权重和为 1。
    4.  **【硬核点】负载均衡（Load Balancing）**：
        *   思考：如果所有 Token 都选 Expert 0，模型会退化。
        *   实现一个简单的 **Auxiliary Loss（辅助损失）**：统计每个专家被选中的频率，并计算其方差。
*   **Day 17 交付物**：能够输出 `topk_indices` 和 `topk_weights` 的 Router 模块。

#### Day 18：分发器（Dispatcher）—— 性能分水岭
*   **目标**：手撕高效的 Token 分发，告别 `for` 循环。
*   **执行细节**：
    1.  **为什么 `for` 循环不行？**：如果你写 `for i in range(8): result = expert[i](x[indices])`，你会触发 8 次 GPU Kernel Launch，显存带宽利用率极低。
    2.  **手撕 `scatter/gather` 逻辑**：
        *   使用 `torch.zeros_like` 准备一个巨大的输出画布。
        *   通过 `indices` 对 Token 进行重新排列（Reordering），将发往同一个专家的 Token 拼成一个大 Batch。
        *   **【关键函数】**：研究 `torch.index_select` 和 `index_add_`。
    3.  **合并计算**：尝试实现“专家并行”的直觉——虽然逻辑上是 8 个专家，但在代码里能否通过 `view` 和矩阵拼接，用一次大的矩阵乘法完成所有激活专家的计算？
*   **Day 18 交付物**：不带 `for` 循环的 `moe_forward` 逻辑。

#### Day 19：集成、校验与稀疏度分析
*   **目标**：拼装完整模型并进行 TDD 测试。
*   **执行细节**：
    1.  **完整集成**：将 Day 14-16 的 `SwiGLUFFN` 实例化 8 次，放入 `nn.ModuleList`。
    2.  **端到端测试**：
        *   **形状校验**：输入 $[B, S, D]$，输出必须还是 $[B, S, D]$。
        *   **稀疏度校验**：统计单次 Forward 中有多少比例的参数被跳过（理论上 75% 的参数不参与计算）。
    3.  **【逻辑学家专项】分析**：
        *   输入两组语义完全不同的句子（如“数学证明”和“文学创作”）。
        *   观察 Router 选中的专家索引是否有明显的偏好。
*   **Day 19 交付物**：完成 `moe_layer.py` 及其对应的单元测试报告。

### 💡 详细指导：如何开始下一步？

既然你现在要开始手撕 `moe_layer.py`，我建议你按以下**伪代码逻辑**开始搭建你的第一步：

#### 1. 定义结构
```python
class MoELayer(nn.Module):
    def __init__(self, dim, hidden_dim, num_experts=8, top_k=2):
        super().__init__()
        # 实例化 8 个专家
        self.experts = nn.ModuleList([
            SwiGLUFFN(dim, hidden_dim) for _ in range(num_experts)
        ])
        # 路由层
        self.router = nn.Linear(dim, num_experts, bias=False)
        self.top_k = top_k
```

#### 2. 实现路由决策（Day 17 的重点）
在 `forward` 中，你首先要得到：
*   `logits = self.router(x)`  -> [Batch*Seq, num_experts]
*   `weights, indices = torch.topk(logits, self.top_k, dim=-1)`
*   `weights = F.softmax(weights, dim=-1, dtype=torch.float32).type_as(x)`

#### 3. 手撕“分发”的心理建设
不要试图一开始就写出最完美的向量化代码。**第一步建议：**
先用最笨的 `for` 循环跑通逻辑，确保 `top_k` 的权重能够正确加权回传。

**面试官最看重的点（必考）：**
*   **Q：** “既然只有两个专家被激活，为什么 MoE 训练时还是很占显存？”
*   **A：** “因为虽然计算是稀疏的，但 8 个专家的参数全部都在显存里（除非用专家并行/卸载）。而且 Router 的梯度需要传回整个网络。”
